# 需求规格文档 (Product Requirements Document)

## 1. 项目概述

**应用名称**：围棋对弈 (Weiqi Game)  
**目标平台**：Android (API 24+)  
**核心定位**：支持本地单机对战与局域网 P2P 联机的轻量级围棋应用

---

## 2. 核心玩法需求

### 2.1 游戏模式

| 模式 | 描述 | 优先级 |
|------|------|--------|
| **本地单机** | 两名玩家在同一设备上交替落子，黑先白后 | P0 |
| **局域网 P2P** | 通过 WiFi 局域网发现对手，建立点对点连接对战 | P0 |

### 2.2 对局流程

```
[模式选择] → [游戏准备] → [对局中] → [终局结算]
                ↓
        局域网模式：NSD 发现/广播 → TCP 握手 → 颜色分配
```

### 2.3 基础规则

- **棋盘规格**：标准 19×19 路棋盘（可扩展支持 13×13、9×9）
- **落子规则**：黑棋先行，双方交替，一次一手
- **计时规则**：V1 版本暂不支持计时，但架构预留扩展接口

## 3. 围棋规则边界细化

### 3.1 气的计算 (Liberty/Atari)

- **定义**：与棋子相邻（上下左右）的空交叉点数量
- **实现要求**：
  - 使用 BFS 遍历相连同色棋子群（Block）
  - 汇总该棋子群周围所有空交叉点
  - 气数为 0 时触发提子

### 3.2 提子 (Capture)

- **触发条件**：落子后，相邻对方棋子群气数变为 0
- **执行逻辑**：
  - 移除该对方棋子群的所有棋子
  - 记录提子数量（V1 仅用于显示，V2 支持贴目）
  - 动画反馈：被提棋子淡出或缩小消失

### 3.3 自杀步/禁着点 (Self-capture/Suicide)

- **判定标准**：
  - 落子后，该位置所属己方棋子群气数为 0
  - 且未造成任何对方棋子群气数变为 0
- **应用表现**：
  - UI 层：落子位置显示红色闪烁提示
  - 逻辑层：拒绝该落子，保持棋盘状态不变
  - 提示文案："禁着点：自杀步"

### 3.4 打劫 (Ko) - 同形重复禁止

- **判定标准**：
  - 当前落子提子后，棋盘状态与历史某状态完全一致
  - 或更严格：仅检查上一步状态（简化版打劫规则）
- **应用表现**：
  - UI 层：落子位置显示黄色警告提示
  - 逻辑层：拒绝该落子
  - 提示文案："禁着点：打劫"

## 4. 交互需求

### 4.1 棋盘自适应

- **需求**：棋盘需适配不同屏幕尺寸（手机/平板）
- **实现策略**：
  - 使用 `Canvas` 绘制，基于容器尺寸动态计算格线间距
  - 保持 1:1 宽高比，留边距放置坐标标注
  - 最小触摸目标：每个交叉点响应区域 ≥ 48dp

### 4.2 边缘点击判定

- **问题**：屏幕边缘交叉点难以精确点击
- **解决方案**：
  - 扩展点击响应区域至棋盘外 50% 格距
  - 点击坐标映射到最近交叉点（曼哈顿距离判定）
  - 点击位置预览：手指按下时显示半透明棋子

### 4.3 非自己回合点击拦截

- **单机模式**：无需拦截，双方共用设备
- **P2P 模式**：
  - **拦截逻辑**：非当前玩家回合时，点击棋盘无响应
  - **反馈机制**：显示 Toast "等待对方落子..."（首次点击）或静默忽略（重复点击）
  - **视觉提示**：当前回合方在屏幕顶部高亮显示

## 5. UI 界面需求

| 界面 | 功能描述 |
|------|----------|
| **主菜单** | 模式选择（单机/联机）、设置入口、规则说明 |
| **游戏界面** | 棋盘（Canvas）、当前回合指示、提子计数、认输按钮 |
| **联机大厅** | 创建房间/加入房间、设备发现列表、连接状态 |
| **结算界面** | 胜负判定（数子法/填空法简化版）、再来一局/返回菜单 |
